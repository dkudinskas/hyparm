Compiling the code

Introduction

A cross-toolchain is required for building the hypervisor. Cross-toolchains create and work with executable code for platforms different from the host platform on which the toolchain runs. They can be custom built using crosstool-ng. Ready-to-use toolchains can be downloaded from Mentor Graphics.

The cross-toolchain prefix must be set in the CROSS_COMPILE environment variable: 

# export CROSS_COMPILE=/opt/arm-none-eabi-4.4/bin/arm-none-eabi-

The hypervisor can be customized at compile time using make config. The default configuration can be restored at any time using make defconfig. Next, the hypervisor can be built using make.

The build system supports parallel build with make -jN where N denotes the number of parallel jobs.

Example build steps

A typical example of the steps to compile the hypervisor:

# export CROSS_COMPILE=/opt/arm-none-eabi-4.4/bin/arm-none-eabi-
# make config
# make
